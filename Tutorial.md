# Punkdom 使用指南

这份指南按第一次使用 Punkdom 的顺序来写。你可以一边打开页面一边跟着做：

<http://localhost:5174>

Punkdom 是一个本地优先的 AI 创作工作台，适合写小说、整理设定、和 AI 一起推进长篇创作，也可以把故事变成可互动的剧情体验。

## 1. 先认识界面

打开页面后，先看三个区域：

- 顶部栏：切换「写作模式 / 互动模式」，也可以切换主题和中英文界面。
- 左侧菜单：进入工作台、设定集、叙事编排、版本管理、书籍管理、技能点、智能体、自动化和设置。
- 右侧创作 Agent：和 AI 对话、让 AI 写大纲、续写、润色、检查一致性。

底部状态栏会显示当前版本、运行状态和正在使用的模型。如果这里显示了模型名，说明当前模型配置已经被 Punkdom 识别。

## 2. 配好 AI 模型

第一次使用前，建议先进入「设置」检查模型。

1. 点击左侧「设置」。
2. 在「公共配置」里找到「模型」。
3. 填入 OpenAI 兼容接口的信息：
   - `API Key`
   - `Base URL`
   - `模型`
4. 如果你有多个模型，可以点「添加模型」保存多模型配置。
5. 点击右上角「保存」。

常见用法是把一个速度快的模型用于日常讨论，把一个更强的模型用于生成正文、整理设定或做一致性检查。

## 3. 创建或选择一本书

点击左侧「书籍管理」进入书架。

你有三种开始方式：

- 「新建书籍」：从空白项目开始。
- 「导入现有小说」：把已有正文导入 Punkdom 管理。
- 「导入酒馆角色卡」：从角色卡开始做互动故事或角色驱动创作。

选择一本书后，顶部会显示当前书籍名称。Punkdom 会在本地工作区保存创作文件和 `.punkdom/` 状态目录，你的正文、设定、会话和版本都保存在本机。

## 4. 写作模式：完成第一轮创作

顶部选择「写作模式」，左侧点「工作台」。

写作工作台通常由三部分组成：

- 左侧：作品目录、项目文件和全局搜索。
- 中间：Markdown 编辑器，用来写正文、灵感和设定文件。
- 右侧：创作 Agent，用自然语言帮你规划、写作和修改。

### 第一步：写下灵感

在左侧目录里打开 `ideas.md`。你可以先写下最粗糙的想法，例如：

```text
我想写一个近未来都市奇幻故事。主角能听见城市的梦，核心冲突是城市正在忘记自己的历史。
```

不用一开始就写完整大纲。Punkdom 更适合边讨论边沉淀。

### 第二步：让创作 Agent 帮你启动新书

在右侧创作 Agent 输入：

```text
请和我一起启动这本新书，先根据 ideas.md 追问我还缺哪些关键设定。
```

Agent 会根据当前作品上下文和你的输入继续追问。你可以像和编辑讨论一样回答它。

### 第三步：生成大纲或下一组细纲

当你对题材、主角、冲突和世界观比较确定后，可以使用右侧快捷按钮：

- 「下一组细纲」：规划接下来一小组章节。
- 「按细纲写下一章」：根据当前细纲生成下一章。
- 「续写下一段」：在当前章节后继续写。
- 「润色当前章」：优化当前打开的章节。
- 「一致性检查」：检查设定、人物状态和前后文是否冲突。

也可以直接在输入框里输入 `/`，选择可用命令或 Skills。

### 第四步：编辑和保存正文

AI 写出的内容可以继续人工修改。编辑器支持自动保存，也可以在「设置 > 写作模式 > 编辑器」里调整：

- 自动保存
- 自动保存延迟
- 章节文件名模板
- 分卷目录模板
- 最大同时打开 Tab 数
- 是否启用草稿流程

如果你想调整阅读体验，可以点编辑器右上角「设置」，修改正文字号、行间距和编辑器背景。

## 5. 设定集：把长期设定沉淀下来

点击左侧「设定集」。

这里适合保存长期稳定的信息，例如：

- 角色身份、人设、关系和能力
- 世界观规则
- 组织、地点、物品
- 互动故事的开场预设
- 写作风格和创作规则

建议把「会反复影响后续创作」的信息放进设定集，把临时想法留在 `ideas.md` 或当前章节讨论里。这样 Agent 后续引用时更清晰，也不容易把一次性的聊天内容当成长设定。

## 6. 叙事编排：告诉 AI 要怎么讲故事

点击左侧「叙事编排」。

叙事编排可以理解成「导演风格」。它会影响：

- 题材倾向
- 角色反应
- 剧情裁定方式
- 节奏推进
- 长期叙事原则

如果你在写悬疑，可以让叙事编排更重视线索、公平性和反转。如果你在写爽文，可以让它更重视目标、压力、升级和即时反馈。

写作模式和互动模式都会用到叙事编排。

## 7. 互动模式：开始一条可玩的故事线

顶部切换到「互动模式」。注意：点击左侧一级菜单不会自动切换模式，只有顶部的「写作模式 / 互动模式」按钮才会切换。

互动模式的核心区域叫「故事舞台」。第一次进入一条新的故事线时，你会看到开场选择：

- 「AI 生成开场」：让 AI 根据当前书籍和设定生成开局。
- 「使用书籍预设」：使用设定集里配置好的开场白。
- 「使用自定义开局」：自己写一段开局，再让 Punkdom 接着推进。

开场生成后，你可以在输入框里写行动，例如：

```text
我检查手机里最后一条未读消息。
```

然后点击「发送」。如果你想让系统给一些行动建议，可以点「选择」。

右侧「故事记忆」会整理当前剧情、角色状态、进行中事项和剧情纪要。互动故事变长后，它能帮助 Agent 记住真正重要的长期状态。

## 8. 版本管理：大胆试写，随时回退

点击左侧「版本管理」。

建议在这些时机手动保存版本：

- 完成一章后
- 大改设定前
- 让 Agent 批量修改文件前
- 导入小说或角色卡后
- 准备尝试一条新剧情路线前

版本管理里可以查看版本历史、对比差异，并在需要时回滚。删除文件前 Punkdom 也会保存版本，方便恢复。

## 9. 技能点、智能体和自动化

这三个模块适合你熟悉基础流程后再使用。

- 「技能点」：把常用写作流程封装成可复用能力，例如固定的大纲方法、审稿流程、风格检查。
- 「智能体」：配置不同 Agent 的模型、提示词和工具权限。
- 「自动化」：让 Punkdom 在特定条件下自动做事，例如章节 Review、资料整理或定期检查。

刚开始不需要先配置很多。先用默认创作 Agent 跑通一本书的开局，再逐步把重复流程沉淀成 Skills 或自动化。

## 10. 推荐的新手流程

如果你不知道从哪里开始，可以按这个顺序做：

1. 进入「设置」，确认模型可以使用。
2. 进入「书籍管理」，新建一本书。
3. 回到「写作模式 > 工作台」，打开 `ideas.md` 写下灵感。
4. 在右侧创作 Agent 里让它追问并整理新书方向。
5. 把稳定的人设、世界观和规则沉淀到「设定集」。
6. 用「下一组细纲」规划前几章。
7. 用「按细纲写下一章」生成第一章。
8. 人工修改正文，并保存一个版本。
9. 切到「互动模式」，用同一本书的设定生成一条故事线。
10. 根据实际写作习惯，再配置叙事编排、技能点和自动化。

## 11. 使用时的小建议

- 和 Agent 对话时，尽量说明目标、范围和限制，例如「只规划前三章，不要写正文」。
- 重要设定不要只留在聊天里，确认后放进设定集或项目文件。
- 大改前先保存版本。
- 如果 AI 方向跑偏，直接指出问题并要求它先复述当前设定，再继续创作。
- 长篇项目不要一次让 AI 生成太多章，先做短期细纲，再逐章推进。
- 写作模式适合生产正文和资料；互动模式适合体验剧情分支、角色行动和即时反馈。

完成以上步骤后，你就已经掌握了 Punkdom 的基本闭环：整理灵感、沉淀设定、编排叙事、生成正文、互动推进、保存版本。
